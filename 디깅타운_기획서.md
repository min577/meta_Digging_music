### Claude Code 빌드용 프롬프트 기획서 · v0.1

> 레퍼런스: [focustown.app](https://www.focustown.app/) — "함께 공부"를 **"함께 디깅"**으로 전환

## 0. 한 줄 정의

> **"혼자 듣지 마세요." 전 세계 사람들과 같은 곡을 동시에 들으며, 내 캐릭터로 취향을 표현하고, 취향이 비슷한 사람을 발견하는 음악 디깅 메타버스.**

Focustown이 "공부 라이브 룸"이라면, DigTown은 **"리스닝 파티 라이브 룸"**이다.
공부량 대신 **디깅(곡 발견)**, 도서관 대신 **무드별 음악 공간**, "무엇을 공부 중"이 아니라 **"무엇을 듣는 중 / 어떤 취향"**이 사람을 보여준다.

## 1. 핵심 차별점 (북극성)

| | Focustown | **DigTown** |
|---|---|---|
| 행동 | 함께 집중/공부 | 함께 듣기 + 곡 발견(디깅) |
| 룸 테마 | 도서관/카페/비행기 | 무드/장르 공간 (Late Night Jazz, City Pop Train…) |
| 사람 표현 | 국기 + "공부 중 과목" | **캐릭터가 취향을 시각화** (장르→외형·이펙트) |
| 매칭 원리 | 인접/랜덤 | **취향 유사도 우선** (KHUX 인사이트) |
| 게이미피케이션 | 아바타·포스트카드 | 아바타 진화 + 아티스트 랭킹 + 디깅 퀘스트 |

**가장 중요한 설계 원칙 (절대 흔들리지 말 것):**
> KHUX 리서치 결과 — **음악 취향 유사도가 개인적 친밀도보다 압도적으로 중요(10/10).**
> → 룸 추천, 사람 추천, 매칭의 **1순위 정렬 기준은 항상 "취향 일치도"**다. 친구/팔로우는 2순위.

## 2. 기술 스택 (Vercel 최적화)

| 영역 | 선택 | 이유 |
|---|---|---|
| 프레임워크 | **Next.js 14 (App Router) + TypeScript** | Vercel 1급 시민, SSR/RSC |
| 스타일 | **Tailwind CSS** + Framer Motion | Focustown식 아기자기한 톤 + 캐릭터 애니메이션 |
| 상태관리 | **Zustand** | 가볍고 룸 상태 관리에 적합 |
| 백엔드/DB | **Supabase** (Postgres + Auth + Realtime) | Vercel 연동 쉬움, 무료티어, **Realtime이 리스닝 파티 동기화 핵심** |
| 인증 | Supabase Auth (소셜 로그인: Google/Apple/Kakao) | 온보딩 마찰 최소화 |
| **음악 재생** | **YouTube IFrame Player API** | MVP 현실적 선택 (저작권·동기화). Spotify SDK는 Phase 3 |
| 실시간 동기화 | Supabase Realtime (broadcast/presence) | 곡 위치·참가자·반응 동기화 |
| 배포 | Vercel + Supabase | GitHub 연동 자동 배포 |

> **음악 소스 주의 (반드시 인지):** 임의 음원의 동기화 스트리밍은 저작권 이슈가 있다. MVP는 **YouTube IFrame API**로 "같은 영상을 같은 위치에서 재생"하는 워치파티 방식이 가장 안전·현실적. 서버는 음원을 저장하지 않고 "현재 videoId + 시작 timestamp"만 broadcast → 각 클라이언트가 seek 동기화. Spotify/Apple Music 연동은 사용자 본인 계정 기반으로 Phase 3에서 검토.

> 민우님은 Python 선호지만, Vercel 배포·실시간·프런트가 핵심이라 **Next.js/TS가 정답**. 추천 알고리즘 등 ML성 로직이 무거워지면 별도 **FastAPI 마이크로서비스**(Railway/Fly.io)로 분리 가능.

## 4. 핵심 기능 명세

### 4.1 온보딩 (취향 시드 수집)
- 캐릭터 생성: 베이스 캐릭터 + 기본 의상 선택 (이미지4 캐릭터 에셋 활용)
- **자주 듣는 상황 선택**: "공부할 때 / 운동할 때 / 자기 전 / 출퇴근…" (룸 추천 시드)
- **취향 3곡 선택**: 검색하여 3곡 등록 → **취향 벡터 초기값** 생성
- 결과: `user.taste_vector` (장르·무드·BPM·아티스트 임베딩) 초기화

### 4.2 리스닝 파티 룸 (메인 화면) ⭐
앱의 심장. 아래 "5. 리스닝 파티 설계" 참조.
- 상단: 현재 곡 정보 + 앨범아트 + 진행바 (동기화)
- 중앙: **공간(무드 배경) + 참가자 캐릭터들** (각자 취향 시각화, 음악에 맞춰 반응)
- 우측/하단: 큐(다음 곡) · 채팅 · 실시간 반응(하트/이모지)
- 인터랙션: 곡 좋아요, **디깅함 저장**, 곡 제안(협업 큐 모드), 캐릭터 감정표현

### 4.3 캐릭터 / 아바타 시스템 (게이미피케이션 코어)
- **취향 표현**: 듣는 장르에 따라 캐릭터 외형·이펙트 실시간 변화
  - 예: Jazz→중절모/세피아 톤, City Pop→네온/선글라스, Lo-fi→후드/빗방울, Metal→가죽자켓/번개
- **진화/육성**: 청취량·발견곡 수·연속 출석에 따라 레벨업 → 외형 단계 진화
- **음악 반응**: BPM에 맞춰 캐릭터가 까딱이거나 춤 (Framer Motion 루프 애니메이션)
- 꾸미기: 상점 아이템으로 커스터마이즈 (이미지4 캐릭터 그리드)

### 4.4 아티스트별 청취 랭킹
- 룸/글로벌/친구 단위 랭킹
- "당신은 [아티스트] 상위 3% 리스너" 식 배지
- 같은 아티스트 상위 리스너끼리 **취향 매칭 추천** (인사이트 적용)

### 4.5 디깅 퀘스트 (후궁선택게임 / 방탈출 요소)
- **무드 선택 퀘스트**: "오늘 밤 어떤 기분?" → 선택지 분기 → 큐레이션 룸으로 안내 (후궁선택 방식)
- **협동 미션 / 방탈출**: 룸 단위 협동 미션 — "장르 맞히기", "숨은 명곡 찾기", "5명이 같은 곡 좋아요 누르면 보상 해금"
- **데일리 퀘스트**: 새 장르 1곡 발견, 새로운 룸 입장 등 → 재화/배지

### 4.6 재화 & 상점
- 코인(구매·보상), 디깅 포인트(활동 보상)
- 상점: 아바타 의상 / 룸 테마 / 부스트(랭킹 가속 등) / 진화 아이템
- 구매 확인 모달

## 5. 리스닝 파티 설계 — 언제 · 어떻게 · 누가 ⭐

> 민우님이 명시적으로 던진 질문. MVP 기준으로 정리.

### 🕐 언제 (When)
1. **Always-on 라이브 룸 (MVP 기본)** — Focustown 방식. 테마 룸이 24/7 열려있고 아무 때나 입장. 진입장벽 0.
2. **예약형 리스닝 파티 (Phase 2)** — 호스트가 시간 지정. "오늘 밤 22:00, 시티팝 디깅". 푸시 알림 → 기대감·리텐션.
3. **정기 큐레이터 파티 (Phase 3)** — 매주 고정 시간, 큐레이터/아티스트 진행. 이벤트성.

### 🎧 어떻게 (How) — 동기화 재생 메커니즘
- 룸은 **공유 큐(queue)**와 **재생 상태(현재 videoId + serverStartTime)**를 가짐
- 서버(Supabase Realtime)가 재생 상태를 broadcast → 각 클라이언트가 `(now - serverStartTime)` 만큼 seek → **동기화 청취**
- **큐 운영 모드 3종:**
  - **🎙 DJ 모드**: 호스트만 곡 추가/스킵 (큐레이션 파티)
  - **🤝 협업 큐 모드**: 참가자가 곡 제안 → 좋아요 많은 순 자동 정렬 → **디깅의 핵심 재미**
  - **📻 라디오 모드**: 룸 태그 기반 자동 재생 (무인 운영)
- **상호작용**: 실시간 하트/이모지 반응, 채팅, "디깅함 저장", 캐릭터 감정표현
- 곡 종료 시: 다음 곡 자동 + **발견곡 카드** 누적 → 세션 종료 시 요약

### 👥 누가 (Who) — 매칭 (인사이트 적용)
- **룸 추천**: `cosine_similarity(내 taste_vector, 룸의 곡 분포 vector)` → **"취향 일치순"** 정렬이 기본 정렬
- **룸 내 사람**: 캐릭터가 취향을 시각화 → 둘러보다 비슷한 사람 발견 → 프로필 보기 / 친구 추가
- **호스트**: 누구나 룸 생성. 공개 / 비공개(링크) / 친구한정
- **정원 타입**:
  - 소규모 디깅방 (~10명): 친밀한 곡 공유, 협업 큐 활발
  - 대형 라이브방 (수백~천+): 큐레이터 파티, DJ 모드
- **친구는 2순위**: 친구가 있는 방을 상단 노출하되, **취향 일치도가 정렬의 우선**

## 6. 데이터 모델 (Supabase / Postgres 초안)

```sql
-- 사용자
users (
  id, handle, country,
  taste_vector jsonb,        -- 장르/무드/BPM/아티스트 임베딩
  coins int, digg_points int,
  level int, created_at
)

-- 캐릭터/아바타
characters (
  id, user_id, base_type,
  equipped jsonb,            -- 장착 의상
  evolution_stage int
)

-- 무드 공간(장소)
locations ( id, name, theme, bg_asset, mood_tags text[] )

-- 룸 (리스닝 파티)
rooms (
  id, location_id, host_id,
  title, visibility,         -- public | unlisted | friends
  queue_mode,                -- dj | collab | radio
  taste_vector jsonb,        -- 룸 곡 분포 (추천용)
  current_track jsonb,       -- {videoId, startedAt}
  capacity int, is_live bool,
  scheduled_at timestamptz   -- 예약형(Phase 2)
)

-- 룸 참가/프레즌스
room_members ( room_id, user_id, seat_index, joined_at )

-- 큐
room_queue ( id, room_id, video_id, title, artist, suggested_by, likes int, position int )

-- 디깅함 (저장곡)
diggs ( id, user_id, video_id, title, artist, genre, discovered_in_room, created_at )

-- 청취 이력 (랭킹/취향 벡터 업데이트)
listen_events ( id, user_id, video_id, artist, genre, listened_seconds, created_at )

-- 배지/퀘스트
badges ( id, user_id, badge_type, earned_at )
quests ( id, user_id, quest_type, progress, completed_at )

-- 친구/초대
friendships ( user_id, friend_id, status )
```

> 추천 정렬: `taste_vector` 코사인 유사도. MVP는 장르 비율 벡터(예: `{jazz:0.4, citypop:0.3, lofi:0.3}`)로 충분. 임베딩 고도화는 후순위.

## 7. Claude Code 단계별 빌드 플랜

> 각 Phase를 Claude Code에 **하나씩** 주문. 완료·배포 확인 후 다음으로.

### Phase 0 — 셋업
```
Next.js 14 (App Router) + TypeScript + Tailwind 프로젝트를 생성하고,
Supabase 클라이언트를 연결해줘. GitHub에 푸시하고 Vercel에 배포 가능한
상태로 만들어줘. 환경변수(.env.local)에 SUPABASE_URL, SUPABASE_ANON_KEY 사용.
하단 탭 네비게이션(홈/상점/친구/프로필) 골격과 라우팅을 잡아줘.
```

### Phase 1 — 인증 + 온보딩
```
Supabase Auth(Google 소셜 로그인)를 붙이고, 온보딩 플로우를 만들어줘:
(1) 캐릭터 베이스 선택, (2) 자주 듣는 상황 선택, (3) 취향 3곡 검색·등록.
등록한 3곡으로 users.taste_vector(장르 비율 jsonb) 초기값을 생성해 저장.
디자인 톤은 focustown.app처럼 따뜻하고 아기자기하게(베이지/크림 배경, 둥근 카드).
```

### Phase 2 — 홈 탐색 + 룸 목록
```
홈 화면을 만들어줘. focustown의 'Popular live rooms' 캐러셀처럼,
무드 공간(장소) 카드 + 인기 라이브 룸 리스트를 보여줘.
룸 리스트는 내 taste_vector와 룸 taste_vector의 코사인 유사도로
'취향 일치순' 정렬을 기본값으로 해줘. 룸 카드에 참가자 캐릭터 미리보기,
현재 인원, LIVE 뱃지 표시. 룸 탐색 필터(테마별/취향 일치순) 포함.
```

### Phase 3 — 리스닝 파티 룸 (핵심)
```
리스닝 파티 룸 화면을 만들어줘. YouTube IFrame Player API로 곡을 재생하고,
Supabase Realtime broadcast로 (현재 videoId + startedAt)을 동기화해서
모든 참가자가 같은 곡·같은 위치를 듣게 해줘. 화면 구성:
- 상단: 현재 곡/앨범아트/진행바
- 중앙: 무드 배경 + 참가자 캐릭터들(자리 배치)
- 우측: 큐 + 채팅 + 실시간 하트/이모지 반응
큐 모드는 우선 'collab(협업 큐)': 참가자가 곡 제안→좋아요순 자동 정렬.
'디깅함 저장' 버튼, 곡 좋아요 기능 포함. presence로 입퇴장 동기화.
```

### Phase 4 — 캐릭터 시스템
```
캐릭터가 현재 듣는 곡의 장르에 따라 외형/이펙트가 바뀌게 해줘
(예: jazz→세피아·중절모, citypop→네온, lofi→후드·빗방울).
BPM에 맞춰 Framer Motion으로 까딱이는 idle 애니메이션 추가.
청취량/발견곡 수에 따른 레벨업·진화 단계도 구현.
```

### Phase 5 — 디깅함 + 취향 리포트 + 아티스트 랭킹
```
프로필/마이페이지: 디깅함(저장곡 컬렉션 그리드), 취향 리포트(장르 분포 차트,
시간대 패턴), 아티스트별 청취 랭킹('상위 N% 리스너' 배지)을 만들어줘.
listen_events 기반 집계. 같은 아티스트 상위 리스너 추천 섹션 포함.
```

### Phase 6 — 디깅 퀘스트 (게이미피케이션)
```
디깅 퀘스트를 추가해줘: (1) 무드 선택 퀘스트(후궁선택 방식 분기→큐레이션 룸 안내),
(2) 룸 협동 미션(5명이 같은 곡 좋아요→보상 해금), (3) 데일리 퀘스트.
완료 시 코인/디깅포인트/배지 지급.
```

### Phase 7 — 상점 + 친구 + 예약형 파티
```
상점(아바타 의상/룸 테마/부스트, 구매 확인 모달), 친구(검색·초대 링크·추천 코드),
예약형 리스닝 파티(scheduled_at + 알림)를 추가해줘.
```
